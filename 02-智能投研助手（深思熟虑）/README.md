# 智能投研助手（深思熟虑）

---

## 一、项目背景与目标

在投资研究领域，分析师需要整合大量市场数据、行业信息和宏观经济指标，进行多维度分析才能形成投资观点。传统的人工研究方法耗时耗力，且容易出现信息遗漏或分析偏差。

本项目基于LangGraph框架实现了"深思熟虑型"智能体，模拟专业投资研究员的思维方式，通过五阶段流程系统化地完成从数据收集到报告生成的全过程，为投资研究提供智能化辅助。

---

## 二、项目简介

本项目（INTELLIGENT RESEARCH ASSISTANT - DELIBERATIVE）是一个基于LangGraph和LangChain的智能投研助手系统。它通过五阶段深思熟虑流程，模拟专业投资研究员的思维模式，能够系统化地收集市场数据、构建分析模型、生成多个候选方案、选择最优策略，并输出专业的投资研究报告。

**适用人群：**
- 投资分析师、研究员
- 基金经理、投资顾问
- 金融科技从业者
- 金融AI/NLP学习者
- 投资机构研究人员
- 任何需要进行投资研究的专业人士

## 四、核心功能与特性

- **五阶段深思熟虑流程**：感知→建模→推理→决策→报告，完整模拟专业投资研究流程
- **多候选方案生成**：推理阶段生成3个不同的投资分析方案，涵盖不同投资思路
- **智能决策机制**：基于方案置信度、优缺点等综合评估，选择最优投资策略
- **专业报告生成**：自动生成结构完整、逻辑清晰的投资研究报告
- **状态管理**：基于LangGraph的状态图管理，确保流程可控和可追溯
- **错误处理与重试**：每个阶段都有异常处理和重试机制（最多3次），提高系统稳定性
- **日志记录**：完整的日志系统，便于调试和问题追踪
- **可视化流程图**：自动生成Mermaid流程图，直观展示工作流

## 五、系统原理与技术细节

### 1. 深思熟虑智能体架构
系统采用LangGraph的状态图（StateGraph）架构，通过节点和边的方式组织工作流，每个阶段独立执行并更新共享状态。

### 2. 五阶段处理流程

**阶段1：感知（Perception）**
- 收集市场概况和最新动态
- 获取关键经济和市场指标
- 整理近期重要新闻
- 分析行业趋势（至少3个细分领域）

**阶段2：建模（Modeling）**
- 基于感知数据构建内部世界模型
- 评估当前市场状态
- 判断经济周期阶段
- 识别主要风险因素（至少3个）
- 发现潜在机会领域（至少3个）
- 分析市场情绪

**阶段3：推理（Reasoning）**
- 基于世界模型生成3个不同的投资分析方案
- 每个方案包含：投资假设、分析方法、预期结果
- 评估每个方案的置信度（0-1）
- 分析方案的优势（至少3点）和劣势（至少2点）

**阶段4：决策（Decision）**
- 评估所有候选分析方案
- 基于假设、方法、结果、置信度和优缺点选择最优方案
- 形成投资论点和支持证据
- 进行风险评估
- 给出具体的投资建议和时间框架

**阶段5：报告（Report Generation）**
- 整合所有阶段的数据和决策结果
- 生成完整的投资研究报告
- 报告包含：标题摘要、市场背景、核心观点、详细论证、风险因素、投资建议、时间框架和预期回报

### 3. 状态管理（State Management）
系统使用TypedDict定义智能体状态，包括：
- 输入参数：研究主题、行业焦点、时间范围
- 处理状态：感知数据、世界模型、推理方案、选定方案
- 输出结果：最终研究报告
- 控制流：当前阶段、错误信息

### 4. 重试与错误处理机制
- 每个阶段的LLM调用都包含最多3次重试
- 遇到错误时记录日志并保持或回退到适当阶段
- 确保数据依赖关系正确（如建模需要感知数据）
- 用户友好的错误提示

## 八、快速开始（小白也能用）

### 1. 安装依赖
```bash
pip install langchain langchain-community langgraph dashscope pydantic
```

### 2. 配置API密钥
设置环境变量（推荐方式）：
```bash
export DASHSCOPE_API_KEY="your-api-key-here"
```

或在代码中直接配置（见第40行）。

### 3. 运行智能投研助手
```bash
cd "02-智能投研助手（深思熟虑）"
python deliberative_research_langgraph.py
```

### 4. 交互式输入
系统会提示输入：
- **研究主题**：例如"新能源汽车行业投资机会"
- **行业焦点**：例如"电动汽车制造、电池技术"
- **时间范围**：短期/中期/长期

### 5. 查看结果
- 终端显示五阶段执行过程
- 最终显示完整研究报告
- 报告自动保存为 `research_report_YYYYMMDD_HHMMSS.txt`

## 十三、常见问题（FAQ）

**Q1：系统需要联网吗？**
- 是的，需要调用通义千问API，需要稳定的网络连接。

**Q2：API调用会产生费用吗？**
- 通义千问API有免费额度，超出部分按使用量计费。建议关注API使用情况。

**Q3：如何提高分析深度？**
- 可以在提示模板中增加更具体的要求
- 调整推理阶段生成的方案数量（当前为3个）
- 在输入时提供更详细的研究主题和行业焦点

**Q4：可以用于其他领域的研究吗？**
- 可以！只需修改提示模板中的领域相关描述，即可应用于其他行业或领域的研究。

**Q5：报告生成失败怎么办？**
- 检查网络连接是否正常
- 确认API密钥是否有效
- 查看日志文件定位具体错误
- 可以尝试重新运行，系统有重试机制

**Q6：如何集成到现有系统？**
- 可以作为模块导入：`from deliberative_research_langgraph import run_research_agent`
- 可以包装为RESTful API服务
- 可以集成到更大的投资研究平台

## 十四、性能与优化建议

### 1. 性能考虑
- 每次研究需要调用5次LLM API
- 每次调用最多重试3次，最长执行时间可能较长
- 建议在网络稳定时使用

### 2. 优化建议
- 可以考虑添加缓存机制，避免重复研究相同主题
- 对于批量研究，可以考虑异步处理
- 可以添加进度条显示更友好的进度信息
